# Plotterbench QA test cards

A versioned suite of diagnostic SVGs for testing plotter/protocol compatibility.
Generated by [`generate.py`](generate.py). Plot the **same** cards **identically**
on every machine — sameness is the whole point of a diagnostic.

> Full procedure (when to run what, how to grade, what to do per new machine):
> see `~/Projects/journal/projects/plotterbench/qa/README.md`.

## Why purpose-built cards instead of "plot a nice SVG"

Random art has no ground truth — when it looks wrong you can't tell *what's*
wrong, and you can't compare two machines. Each card here isolates **one**
compatibility dimension so a failure points at a specific cause. The reasoning
splits "compatibility" into three independent questions:

1. **Protocol correctness** — are the right command bytes sent for this firmware
   (EBB vs DrawCore vs vanilla Grbl)? The SVG barely matters; card 00 covers it.
2. **SVG-pipeline fidelity** — does the input geometry come out on paper as drawn?
   Mostly *your software*, machine-independent. Cards 03 / 04 / 08.
3. **Physical/mechanical reality** — calibration, backlash, ringing, lost steps,
   bed extent, endurance. The only part that truly needs a pen on paper.
   Cards 01 / 05 / 06 / 07 / 09.

Questions 1 and 2 are largely testable in software (see the headless capture
harness, backlog task 31) *before* a machine is in hand.

## The cards

| # | File | Isolates | Grade by |
|---|------|----------|----------|
| 00 | `00-smoke.svg` | connect · pen up/down · move · rough scale · orientation | eyeball — run first, ~30 s |
| 01 | `01-calibration-ruler.svg` | dimensional accuracy, steps/mm, X vs Y scale | **ruler/calipers**: 100 mm = 100 mm? circle round not oval? |
| 02 | `02-orientation-key.svg` | origin, Y-flip, rotation, preview-vs-paper | "L" upright & forward? corners where labelled? |
| 03 | `03-curve-fidelity.svg` | flattening tolerance (curve smoothness) | loupe: facets on small circles/arcs = tolerance too coarse |
| 04 | `04-pen-lift-comb.svg` | pen up/down between paths, servo timing | missing/ghost stroke-starts, drag marks, dropped dots |
| 05 | `05-closing-loop.svg` | lost steps, cumulative drift, return-to-origin | gap at the START/RETURN corner = lost steps |
| 06 | `06-corner-ringing.svg` | accel/decel, overshoot — **replot at 3 speeds** | rounded/overshot corners, ringing echoes |
| 07 | `07-bed-extent.svg` | coordinate clipping, out-of-bounds | reaches all corners? oversized circle clipped/refused gracefully? |
| 08 | `08-parser-matrix.svg` | SVG construct coverage (mostly software) | every cell renders correctly; **fill cell = outline only** |
| 09 | `09-endurance-stress.svg` | USB/buffer stability, memory, long-run drift | completes without stall/disconnect; end registers to start |
| 10 | `10-acceptance-art.svg` | integration gate (**placeholder — replace**) | "would I ship this print?" |

**Card 10 is a placeholder.** Drop in one real, fixed, versioned artwork you'd be
proud to ship and plot it identically on every machine alongside the diagnostics.

## Regenerating

```sh
python3 generate.py                 # A4 portrait (default)
python3 generate.py --page letter   # US Letter
python3 generate.py --page a3
python3 generate.py --only 07-bed-extent --page custom --width 297 --height 210
```

**Card 07 is bed-specific.** Regenerate it per machine with the real drawable
area (`--page custom --width <bed_w> --height <bed_h>`) so the frame sits 1 mm
from the true reachable edge. The other cards are A4 and fit every supported bed.

## Conventions (so the cards stay good diagnostics)

- **1 user unit = 1 mm** (`viewBox` numbers match the mm `width`/`height`). A
  "100 mm" line really measures 100 mm — grade with a physical ruler.
- **Geometry carries the ground truth.** `<text>` labels are reference only; most
  plot pipelines skip text, so every measurable feature is gradeable without them.
- **Strokes only, `fill="none"`.** Card 08 includes a `fill="black"` cell on
  purpose: a plotter must render it as an **outline**, not a solid box.
